## Visão Geral

O Decopilot fornece ferramentas integradas que estão sempre disponíveis (sujeitas a restrições de tarefas/subtarefas). Essas ferramentas permitem gerenciamento de trabalho, acesso ao contexto e coordenação de agentes.

## Categorias de Ferramentas

### Descoberta e Habilitação

Ferramentas para ativar capacidades:

- [**tool_enable**](/pt-br/studio/api-reference/built-in-tools/tool-enable) - Ativar uma ferramenta para uso na tarefa atual

### Gerenciamento de Agentes

Ferramentas para trabalhar com agentes:

- [**agent_search**](/pt-br/studio/api-reference/built-in-tools/agent-search) - Encontrar agentes configurados na organização

### Gerenciamento de Tarefas

Ferramentas para criar e gerenciar trabalho:

- [**subtask**](/pt-br/studio/api-reference/built-in-tools/subtask-run) - Criar subtarefa com contexto isolado

### Interação com o Usuário

Ferramentas para coletar entrada do usuário:

- [**user_ask**](/pt-br/studio/api-reference/built-in-tools/user-ask) - Perguntar ao usuário por entrada ou esclarecimento

### Acesso ao Contexto

Ferramentas para ler recursos e prompts:

- [**read_resource**](/pt-br/studio/api-reference/built-in-tools/resource-read) - Ler documentação/diretrizes do escopo atual
- [**read_prompt**](/pt-br/studio/api-reference/built-in-tools/prompt-read) - Ler templates de prompt do escopo atual

### Execução de Código e Saída

Ferramentas para executar código e processar resultados:

- [**sandbox**](/pt-br/studio/api-reference/built-in-tools/sandbox) - Executar código JavaScript com acesso a ferramentas MCP
- [**read_tool_output**](/pt-br/studio/api-reference/built-in-tools/read-tool-output) - Filtrar saídas grandes de ferramentas usando padrões regex

### Planejamento

Ferramentas para execução estruturada:

- [**propose_plan**](/pt-br/studio/api-reference/built-in-tools/propose-plan) - Propor planos de execução para aprovação do usuário

## Matriz de Disponibilidade

| Ferramenta | Tarefa | Subtarefa | Descrição |
|------|:----:|:-------:|-------------|
| `tool_enable` | ✓ | ✓ | Ativar uma ferramenta para uso na tarefa atual |
| `agent_search` | ✓ | ✗ | Encontrar agentes configurados na organização |
| `subtask` | ✓ | ✗ | Criar subtarefa com contexto limpo |
| `user_ask` | ✓ | ✗ | Perguntar ao usuário por entrada ou esclarecimento |
| `read_resource` | ✓ | ✓ | Ler documentação/diretrizes do escopo atual |
| `read_prompt` | ✓ | ✓ | Ler templates de prompt do escopo atual |
| `sandbox` | ✓ | ✓ | Executar código JavaScript com acesso a ferramentas MCP |
| `read_tool_output` | ✓ | ✓ | Filtrar saídas grandes de ferramentas usando regex |
| `propose_plan` | ✓ | ✓ | Propor planos de execução para aprovação |

<Callout type="info">
  **Tarefas** são iniciadas pelo usuário e têm acesso completo a todas as ferramentas integradas. **Subtarefas** são criadas pelo decopilot e têm acesso restrito para evitar delegação infinita e bloqueio de interações com o usuário. Subtarefas têm um limite de 15 passos (vs 30 para tarefas) e devem retornar um resumo.
</Callout>

<Callout type="tip">
  **Descoberta de ferramentas**: As ferramentas de escopo dos servidores MCP conectados são descobertas implicitamente através das conexões listadas no MCP Virtual do escopo atual — não há uma ferramenta explícita de busca.
</Callout>
