## Duas formas comuns de autenticar

- **Sessão de usuário (login)**: para operar o produto via UI e usar MCP management tools como pessoa.
- **API key**: para integrar um client/agent em produção com permissões explícitas.

## Configuração de auth (self-host)

Em ambientes self-host, a configuração de auth normalmente é feita via `auth-config.json` + variáveis de ambiente (por exemplo, `BETTER_AUTH_SECRET`).

Para detalhes, veja:

- [Local: Docker Compose](/pt-br/mcp-mesh/deploy/local-docker-compose)
- [Kubernetes: Helm Chart](/pt-br/mcp-mesh/deploy/kubernetes-helm-chart)

<Callout type="tip">
  Em produção, trate auth como infra: gere secrets fortes e não comite credenciais em Git.
</Callout>
