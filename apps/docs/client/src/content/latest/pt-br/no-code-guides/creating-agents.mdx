Agents são assistentes conversacionais de IA que podem chamar tools, tomar decisões e manter contexto através de conversas.

## Entendendo Agents

Cada agent:
- **Entende linguagem natural** - Usuários podem conversar com eles naturalmente
- **Chama tools autonomamente** - Decide quais tools usar baseado na situação
- **Segue instruções** - Se comporta de acordo com seu system prompt
- **Mantém contexto** - Lembra do histórico de conversa
- **Tem acesso a resources** - Referencia qualquer resource (tools, documents, workflows, views) usando menções `@`

**Casos de uso comuns:**
- Automação de suporte ao cliente
- Assistentes de produtividade interna
- Auxiliares de análise de dados
- Criação de conteúdo
- Automação de processos

## Criando Agents com Deco Chat

A maneira mais fácil de criar um agent é descrevendo o que você precisa:

**Exemplos de prompts:**
```
Preciso de um agent de suporte ao cliente que possa consultar pedidos e processar reembolsos abaixo de R$500

Crie um agent que ajude minha equipe a analisar dados de vendas e gerar relatórios

Construa um agent escritor de conteúdo que siga nossas diretrizes de marca e possa publicar em nosso CMS

Quero um agent que faça triagem de consultas de clientes e as encaminhe para o especialista certo
```

**O que acontece em seguida:**
1. Deco chat faz perguntas esclarecedoras (Quais tools? Qual personalidade? Quais limites?)
2. Sugere um system prompt baseado nas suas necessidades
3. Recomenda quais tools dar a ele
4. Cria o agent usando `@AGENTS_CREATE`
5. Você testa e itera juntos

**Refine conforme avança:**
```
Deixe o tom mais formal

Adicione a capacidade de criar tickets de suporte

Não deixe processar reembolsos acima de R$250 sem aprovação

Inclua nosso FAQ de produto como contexto
```

Comece com uma ideia vaga e itere em direção a exatamente o que você precisa.

## O Que Acontece Por Baixo dos Panos?

Quando você cria um agent (seja via deco chat ou pela UI), estes são os componentes principais que estão sendo configurados:

### 1. System Prompt (Mais Importante)

O system prompt define as características do seu agent:
- **Comportamento** - Como ele age e responde
- **Personalidade** - Tom e estilo
- **Limites** - O que ele deve/não deve fazer
- **Conhecimento de domínio** - Áreas de expertise

**Exemplo de system prompt:**
```
Você é um agent de suporte ao cliente para Acme SaaS.

Você ajuda com:
- Perguntas sobre cobrança e assinatura
- Problemas de acesso à conta
- Explicações de recursos do produto

Você é amigável, paciente e orientado a soluções.

Você pode processar reembolsos de até R$500. Para valores maiores, abra um ticket 
e informe o cliente que um gerente fará a revisão.

Quando não souber algo, seja transparente e ofereça conectá-lo 
com um especialista.

Nunca faça promessas sobre recursos futuros ou preços personalizados.
```

**O que faz um bom system prompt:**

**Seja específico:**
```
❌ "Você é um assistente útil"
✅ "Você é um agent de suporte técnico que ajuda a debugar problemas de integração de API"
```

**Defina limites claros:**
```
Você pode:
- Responder perguntas sobre o produto
- Processar reembolsos abaixo de R$500
- Agendar chamadas com vendas

Você não pode:
- Fazer promessas sobre recursos
- Compartilhar dados de clientes
- Modificar preços
```

**Mostre, não apenas diga:**
```
Exemplo de interação:

Usuário: "Não consigo fazer login"
Você: "Vou ajudá-lo com isso! Deixe-me verificar o status da sua conta. 
      Qual endereço de email você está usando?"
```

<Callout type="success">
  O system prompt é onde acontece a maioria do ajuste do agent. Invista tempo aqui, tem o maior impacto no comportamento.
</Callout>

### 2. Seleção de Modelo

Escolha o modelo de IA que alimenta seu agent:

| Modelo | Melhor Para | Custo | Velocidade |
|--------|-------------|-------|------------|
| GPT-4 | Raciocínio complexo, tarefas multi-etapas | $$$ | Lento |
| Claude 3.5 Sonnet | Performance balanceada, conversas longas | $$ | Rápido |
| GPT-3.5 Turbo | Tarefas simples, respostas rápidas | $ | Muito Rápido |

**Orientação prática:**
- **Suporte ao cliente** - Claude Sonnet (boa compreensão, rápido)
- **Análise de dados** - GPT-4 (melhor raciocínio)
- **Automação simples** - GPT-3.5 (barato, rápido)
- **Escrita de conteúdo** - GPT-4 (mais criativo)

Você pode mudar modelos a qualquer momento.

### 3. Atribuição de Tools

Dê ao seu agent acesso a tools específicos:

**Estratégia: Comece mínimo**
```
❌ Dar acesso a todos os 50 tools do workspace
✅ Começar com 3-5 tools essenciais, adicionar conforme necessário
```

Muitos tools confundem o agent. Ele não saberá qual usar.

**Exemplo de toolset para suporte ao cliente:**
```
Informações do Cliente:
├─ CUSTOMER_FETCH
└─ CUSTOMER_UPDATE

Cobrança:
├─ INVOICE_FETCH
├─ REFUND_PROCESS
└─ SUBSCRIPTION_UPDATE

Comunicação:
├─ EMAIL_SEND
└─ TICKET_CREATE
```

**No seu system prompt, explique quando usar tools:**
```
Para consultar o histórico de pedidos de um cliente, use @ORDERS_FETCH com o email dele.
Para processar um reembolso abaixo de R$500, use @REFUND_PROCESS com o ID do pedido.
Para reembolsos acima de R$500, use @TICKET_CREATE para escalar para um gerente.
```

Você pode referenciar qualquer resource em system prompts usando menções `@` (tools, documents, workflows, views).

**Atribuindo tools de integrações:**
Na aba Agents, clique em **Tools** → **Adicionar Integração** para dar ao seu agent acesso a tools de integração. Adicione novas integrações ou selecione das instaladas.

<Callout type="warning">
  Tenha cuidado ao dar aos agents tools que deletam dados, cobram dinheiro ou chamam APIs caras. Considere adicionar workflows de aprovação para operações sensíveis.
</Callout>

### 4. Acesso a Documents

Anexe documents que o agent pode buscar e referenciar. Documents podem ser mencionados em threads e system prompts de agents usando `@` (ex: `@prd-do-projeto`, `@faq-de-suporte`).

**O que anexar:**
- **Documentação de produto** - Guias de recursos, FAQs
- **Documentação de processo** - SOPs, diretrizes de escalação
- **Informações da empresa** - Políticas, preços, termos

**Exemplo:**
```
Documents do Agent de Suporte ao Cliente:
├─ Guia de Recursos do Produto.md
├─ FAQ de Cobrança.md
├─ Guia de Troubleshooting.md
└─ Quando Escalar.md
```

O agent automaticamente busca nesses quando relevante. Você não precisa dizer a ele, ele descobre quando precisa de mais informação.

### 5. Configurações Avançadas

Ajuste fino no comportamento (a maioria dos agents funciona bem com padrões):

**Max Steps** (padrão: 15)
- Quantas chamadas de tools o agent pode fazer por resposta
- Maior = pode completar tarefas mais complexas
- Menor = respostas mais rápidas e simples

**Max Tokens** (padrão: 4096-64000 dependendo do modelo)
- Comprimento máximo de resposta
- Ajuste baseado no seu caso de uso (respostas curtas vs. conteúdo longo)

**Temperature** (0.0 - 1.0, padrão: varia por modelo)
- Menor (0.2) = Consistente, determinístico (agents de suporte)
- Maior (0.8) = Criativo, variado (agents de conteúdo)

**Context Window**
- O número de mensagens recentes para manter na memória
- Janela maior = melhor contexto mas maior custo

**Controle de Acesso**
- Controle quem pode interagir e editar este agent

<Callout type="tip">
  Configurações padrão funcionam para a maioria dos casos de uso. Só ajuste se notar problemas específicos.
</Callout>

## Padrões Comuns de Agents

### Agent de Suporte ao Cliente
```
Modelo: Claude Sonnet (rápido, bom em conversação)
Tools: CUSTOMER_FETCH, ORDER_LOOKUP, REFUND_PROCESS, TICKET_CREATE
Documents: FAQ, guia de troubleshooting, política de devolução
Tom: Amigável, paciente, orientado a soluções
Visibilidade: Público
```

### Analista de Dados Interno
```
Modelo: GPT-4 (melhor em queries complexas)
Tools: DATABASE_QUERY, CHART_GENERATE, REPORT_CREATE
Documents: Schema do banco de dados, definições de KPI
Tom: Técnico, preciso
Visibilidade: Workspace
```

### Escritor de Conteúdo
```
Modelo: GPT-4 (mais criativo)
Tools: CONTENT_GENERATE, SEO_ANALYZE, CMS_PUBLISH
Documents: Diretrizes de marca, guia de estilo
Tom: Criativo, envolvente, alinhado com a marca
Visibilidade: Workspace
```

## Testando Seu Agent

1. **Caminho feliz** - Tudo funciona como esperado
2. **Informação faltando** - Agent faz perguntas de follow-up
3. **Erros** - Tools falham, agent lida graciosamente
4. **Casos extremos** - Solicitações incomuns ou inesperadas
5. **Multi-etapas** - Tarefas complexas requerendo múltiplos tools
6. **Limites** - Usuário pede ao agent para fazer algo que não deveria

Abra o agent em uma thread e tente conversas reais. Observe quais tools ele chama e como responde. Refine o system prompt baseado no que vê, este é um processo iterativo.

## Monitorando Uso do Agent

Rastreie performance e uso do agent:

- Vá em **Uso** na sua org
- Veja conversas, mensagens, chamadas de tools, custos por agent
- Filtre threads por agent para ver todas as conversas

Use esses insights para:
- Aprender como outros usam o agent
- Identificar problemas comuns ou confusão
- Descobrir casos de uso inesperados
- Melhorar system prompts baseado em padrões reais

**Lembre-se:** Os melhores agents têm system prompts claros e específicos e toolsets cuidadosamente curados. Comece simples e evolua baseado no uso real.
