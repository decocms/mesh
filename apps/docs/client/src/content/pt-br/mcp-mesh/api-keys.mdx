## Quando usar API keys

Use API keys quando você quer integrar um client/agent em produção com:

- autenticação estável (sem sessão de usuário)
- **permissões explícitas** (least privilege)
- rotação e revogação

## Como funcionam as permissões

No Mesh, API keys são criadas com permissões específicas para tools:

- tools de administração (management)
- e, quando aplicável, permissões para chamadas proxied via agents/connections

## Boas práticas

- Crie uma key por workload (por exemplo: “cursor-team-x”, “agent-prod”, “ci-smoke-test”).
- Comece com permissões mínimas e aumente conforme necessário.
- Faça rotação periódica e revogue keys não usadas.
