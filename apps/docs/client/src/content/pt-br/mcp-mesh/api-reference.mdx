## Endpoints MCP

### Management MCP

- `POST /mcp`

Esse endpoint expõe **management tools** via MCP (orgs, connections, agents, API keys, monitoring, etc.).

### Proxy de connection

- `POST /mcp/:connectionId`

Faz proxy de requisições MCP para uma única connection upstream.

### Agent (MCP virtual)

- `POST /mcp/gateway/:gatewayId` (compatibilidade: o path HTTP ainda usa `gateway`, mas refere-se ao Agent)
- `POST /mcp/gateway` (Agent padrão da org; requer `x-org-id` ou `x-org-slug`)

Faz proxy de requisições MCP para um agent (agregação + estratégia de exposição).

## Endpoints de autenticação (alto nível)

Os endpoints exatos podem variar conforme o provider/configuração, mas em geral o Mesh expõe rotas para:

- login/logout
- callback de OAuth
- sessão do usuário
