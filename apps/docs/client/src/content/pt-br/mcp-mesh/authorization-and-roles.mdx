## O modelo

O Mesh aplica controle de acesso em duas camadas:

- **Roles de organização** (permissões de membros dentro da org)
- **Permissões de tool** (o que uma API key ou sessão de usuário pode chamar)

## Roles (padrão + custom)

Toda organização começa com três roles padrão:

- **Owner**: acesso total (incluindo ações sensíveis, como deletes)
- **Admin**: acesso elevado para operação do dia a dia
- **User**: menor privilégio por padrão

Você também pode criar **roles custom** com permissões granulares (toggles por tool/permissão):

- Vá em **Members** → **Manage Roles**
- Crie uma role nova
- Selecione exatamente quais permissões essa role pode executar

## Permissões de API keys (escopo por tool)

API keys são criadas com permissões explícitas para tools de gerenciamento (e, quando aplicável, para uso de tools proxied).

<Callout type="tip">
  Pense em permissões como infra de produção: comece com least privilege e só conceda o que o client realmente precisa.
</Callout>

## Onde isso aparece no produto

- **Members**: roles e convites
- **API Keys**: criar/rotacionar/revogar chaves e restringir permissões
- **Gateways**: reduzir o surface exposto (include/exclude e estratégias)
