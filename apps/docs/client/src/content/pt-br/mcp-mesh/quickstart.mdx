## 1) Rodar o Mesh

Escolha uma opção:

### Opção A: setup local com um comando (npx)

```bash
npx @decocms/mesh
```

### Opção B: setup local com Docker Compose

```bash
# 1. Configure variáveis de ambiente
# Edite .env e configure BETTER_AUTH_SECRET (obrigatório)
# Gere um secret: openssl rand -base64 32
cp conf-examples/env.example .env

# 2. Configure autenticação
cp conf-examples/auth-config.json.example auth-config.json

# 3. Suba a aplicação
docker compose up -d

# 4. Acesse
open http://localhost:3000
```

**Mais detalhes:** [Local: Docker Compose](/pt-br/mcp-mesh/deploy/local-docker-compose)

### Opção C: rodar a partir do código (source)

```bash
git clone https://github.com/decocms/mesh.git
cd mesh
bun install
bun run dev
```

### Opção D: deploy no Kubernetes (Helm)

```bash
# 1. Gere um secret seguro para autenticação
SECRET=$(openssl rand -base64 32)

# 2. Instale o chart com o secret gerado
helm install deco-mcp-mesh . \
  --namespace deco-mcp-mesh \
  --create-namespace \
  --set secret.BETTER_AUTH_SECRET="$SECRET"

# 3. Aguarde os pods ficarem prontos
kubectl wait --for=condition=ready pod \
  -l app.kubernetes.io/instance=deco-mcp-mesh \
  -n deco-mcp-mesh \
  --timeout=300s

# 4. Acesse via port-forward
kubectl port-forward svc/deco-mcp-mesh 8080:80 -n deco-mcp-mesh
```

A aplicação ficará disponível em `http://localhost:8080`.

**Mais detalhes:** [Kubernetes: Helm Chart](/pt-br/mcp-mesh/deploy/kubernetes-helm-chart)

## 2) Conectar um MCP server

Ao abrir uma organização nova, o Mesh normalmente te leva para:

- **Browse Store** (recomendado), ou
- **Connect MCP Server** direto (URL MCP via HTTP + credenciais, se necessário)

A **Store** é um catálogo de MCP servers que você pode conectar. Por padrão, ela aponta para o **MCP Registry** oficial.

## 3) Criar uma organização e convidar membros

- Vá em **Members**
- Convide pessoas
- Ajuste roles (Admin/User) conforme necessário

## 4) Criar um MCP Gateway (servidor virtual)

- Vá em **MCP Gateways**
- Clique em **Create MCP Gateway**
- Escolha **Include** (somente selecionados) ou **Exclude** (tudo exceto os excluídos)

## 5) Criar uma API key para um cliente

Crie uma API key com o mínimo de permissões necessárias (veja [API Keys](/pt-br/mcp-mesh/api-keys)).

## 6) Validar o monitoring

Use **Monitoring** para confirmar:

- as tool calls estão sendo registradas
- latência e taxa de erro aparecem
- filtros (janela de tempo, tool, connection, gateway) funcionam

<Callout type="tip">
  Um smoke test rápido é chamar uma tool via um gateway (assim você valida agregação + proxy + monitoring).
</Callout>
